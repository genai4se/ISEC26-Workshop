Workshop on GenAI Based Software Engineering

Co-located with ISEC 2026 February 19-21, 2026, IIS Jaipur, India

Workshop on GenAi Based Software Engineering

Objective of Workshop

Use of GenAI for software development has seen increasing maturity in the past 3+ years. Gen AI models are being applied to key activities architectural decision making and reverse engineering activities that are generally activities of senior or seasoned team members in the team. Several context aware tools like Cursor, GitHub Copilot, Amazon Q developer agent (Code Whisperer) and OpenAI Codex claim to have enhanced code development, code completion, test case generation, debugging and issue fixing tasks. Some methods have exploited the instruction tuning and reinforcement learning with feedback. Several more small and large models and tools are exploiting RAG techniques and Agentic AI to enhance the software development tasks including compiling and iteratively working towards a goal such as building an application.

From SE perspective, there is a need to analyze how LLM-based agents can tackle different software development and maintenance tasks, i.e. the entire SDLC. Whereas, from the agent perspective, there is a need to throw light on the basic agentic design components,including planning, memory, perception, action and their roles and collaboration mechanisms, in multi-agent settings.

On a parallel track, the GenAI implementations are also seeing a need for Software Engineering rigor on robustness, reliability, scalability for wider adoption. GenAI models require immense computational power, a scalable and optimized infrastructure ensures scale up or down as needed. Security and Privacy is an emerging concern with GenAI based models. Sufficient software engineering guardrails to prevent data leakage, resilience against cyber attacks including prompt injection attacks will enhance adoption of GenAI in IT systems.

The workshop aims to provide a collaborative platform for researchers and practitioners to delve into the convergence of traditional software methodologies together with Generative AI technologies. By synergizing the strengths of Gen AI, LLM Agentic Frameworks, modeling, and knowledge representation for SDLC, our goal is to define a trajectory toward enhanced software engineering practices. We seek discussion on the following questions:

  1. Architecting, designing, developing and maintaining industry-strength software is a multi-skill, long-drawn activity that cannot be effectively addressed by LLMs alone. What kinds of augmentation make it effective?
  2. LLM is a vast storehouse of general information, but the typical need during SDLC is rather sharply focused. How best to bear local knowledge at scale to get the required focus?
  3. How can Generative AI be effectively utilized for mining and constructing purpose-driven knowledge from software artifacts? Can Generative AI catalyze existing legacy modernization techniques to reduce cost, time and improve correctness?
  4. What are the best practices for maintaining the accuracy, relevance, reliability and quality of Generative AI generated software artifacts?
  5. What are the state-of-the-art techniques, experimental models, methodologies, benchmark datasets and evaluation metrics employed for the usage of LLMs and LLM-based agents in SE applications?
  6. What changes are required in software architecture and design patterns in GenAI based deployments, including Responsible and Ethical use of GenAI?
  7. How can we build reliable and secure GenAI applications? DevOps/DevSecOps have clear boundaries of artifacts for deployment, what challenges or opportunities exist with GenAIOps/LLMOps?
  8. What are the evaluation metrics and studies including on Sustainability with adoption of GenAI in IT systems?
  9. What new skills and roles are required with GenAI and Software Engineering?

The workshop aims to foster interactive discussions, enabling participants to collectively shape the future of advanced software engineering. The inaugural edition in ISEC’24 and the 2nd edition in ISEC’25, had an enormous response, focused on discussing the state of the art and practice in code completion, code generation for satisfying non-functional properties, code translation, code analysis and understanding, code repair, code testing, via a set of invited speakers and submitted and reviewed abstracts of papers. In this (Third) edition, we aim to bring the issues of software engineering in the large and hope to discuss how Generative AI, LLM based Agents and RAG can be exploited together with MDE, KE and RE for effective software engineering. We also aim to bring the Software Engineering practices in GenAI adoption.

Call For Paper And Call For Abstract

The workshop aims to provide a collaborative platform for researchers from academia, industry, and practitioners. Through this platform, we expect to delve into the following topics that are the intersection of Generative AI and Software Engineering.

  • Requirements Engineering and Analysis along with Software Architecture and Design
  • Software Development and Maintenance including refactoring and reengineering
  • Software Verification, Testing and Debugging
  • Legacy Modernization
  • Reverse Engineering from code, documents
  • Human interaction with LLMs
  • Retrieval augmentation techniques for software analysis
  • Agentic frameworks in software engineering
  • Software Engineering architecture and design in GenAI based applications
  • Software Quality including Security & Privacy in GenAI based applications
  • Engineering Sustainable GenAI based Systems
  • Evaluation and Assessment of Agentic AI systems

Submission Information : Papers should be original work in the ISEC’26 ACM Format. Abstract should be original work, in text format, written in English, no more than 500 words describing case studies, interesting experiments, best practices, and lessons learned while applying Generative AI to various SE areas as well as while applying SE practices to engineer GenAI enabled tools and applications

Acceptance criteria : Submissions (Papers, Abstracts and Talks) will be selected based on eminent reviewers from academics and industry. The tentative criteria will be the clarity of articulation of the problem being solved, bringing out the specific need for Generative AI to solve the problem, Software Engineering practices in GenAI and the novelty of the approach.

 
Authors of accepted submissions will receive further instructions for submitting camera-ready presentations. At least one of the authors MUST register for the ISEC conference to present their paper.
